# Стратегия развития и План будущих улучшений (ROADMAP)

Проект **Minecraft Monitoring** достиг высочайшей степени готовности базового функционала уровня Production. 
Уже полностью реализованы: асинхронный пинг (через ReactPHP), бизнес-логика рейтингов, RCON, JWT токены, сброс паролей через SMTP, Система Отзывов, гибкие визуальные темы.

Ниже представлен план последующей массивной экспансии, нацеленной на масштабирование аудитории и Enterprise-уровень (Phases 8 & 9):

---

## 🚀 Phase 8: Gamification Economy & Programmatic SEO

### 1. Ранговая статус-система (Gamification)
Превращаем абстрактные цифры активности в социальную ценность и престиж.
- **Статусы (Ранги)**: Скрытые баллы активности конвертируются в публичные Ранги (Bronze, Silver, Legendary) с цветами ников и приоритетом модерации (строго без вмешательства в силу голоса).
- **Сезонные Лидерборды**: Внедрение "Топ 100 месяца".
- **Анти-Фарм Экономика**: Внедрение diminishing returns на рутинные действия, жесткие дневные лимиты (caps) и progress-бары до следующего статуса. Никакой инфляции.

### 2. Пороговое Программное SEO и SEO Блог
Умная генерация сотен посадочных страниц низкочастотных запросов без "мусорного" контента.
- **Пороговый Генератор Фильтров**: Динамические страницы комбинаций ("Версия + Режим"). Если объем уникального контента и серверов (>=15) достаточен, страница индексируется. Иначе — авто-noindex и canonical на родителя.
- **Шаблонизатор Текстов**: Автоматическая подстановка реальной live-статистики серверов в текста SEO страниц.
- **Коммерческий Блог**: Семантические лонгриды с оглавлениями, H1-H3 разметкой и мощной перелинковкой на сгенерированные SEO-страницы.

## 🔒 Phase 9: Demand-Driven API & Adaptive Security

### 3. Минималистичное Публичное API
Масштабируемый read-only фундамент с защитой от abuse-парсинга.
- **Read-Only Cursor Endpoint**: Один вылизанный эндпоинт `GET /api/v1/servers` с жесткой структурой (data/meta/errors) и супербыстрой cursor-пагинацией.
- **IP-Throttling & Soft-Blocks**: Выдача ключей вручную в ЛК. Автоматическое выявление аномалий трафика (софт-блок нарушителей). Расширение функционала API замораживается до появления реального внешнего спроса.

### 4. Адаптивная Безопасность и Google 2FA
Баланс между высочайшей UX конверсией и банковской защитой зон риска.
- **Soft-Lock (Мягкая Блокировка)**: Google Authenticator не принудителен с ходу. При попадании в зону риска (владение сервером/балансом) начинается отсчет **72-часового грейс-периода** (soft-lock), после которого закрывается доступ только к финансовым разделам панели.
- **Security Service**: Инфраструктурный слой защиты ядра, поддерживающий Backup Codes, логирование IP-входов, с запасом расширяемости до детекта устройств.
