# Стратегия развития и План будущих улучшений (ROADMAP)

Проект **Minecraft Monitoring** достиг высочайшей степени готовности базового функционала уровня Production. 
Уже полностью реализованы: асинхронный пинг (через ReactPHP), бизнес-логика рейтингов, RCON, JWT токены, сброс паролей через SMTP, **Система Отзывов**, гибкие визуальные темы и базовый публичный API.

Ниже представлен план последующей массивной экспансии (Phases 8 & 9):

---

## 🚀 Phase 8: SEO Экспансия и Удержание (Gamification)

### 1. Архитектура 100 Достижений (Геймификация)
Построение мощной системы удержания обычных игроков.
- Разработка системы начисления **Баллов (Points)** на баланс за любые активности.
- Создание словаря на **100 уникальных достижений** (Голосования, Социалки, Чтение SEO-блогов, Оставление отзывов, Добаление аватарок).
- Выдача значков (Badges) в профиль и формирование "Топ пользователей по очкам достижений".

### 2. SEO-Оптимизированный Блог / Гайды (Content Marketing)
Органический трафик — самый дешевый и лояльный. Нам нужен плацдарм для приема трафика из поисковиков.
- Расширение текущей системы новостей до полноценного Блога (`/blog`).
- Написание ЧПУ роутов (SEO Ссылок: `/blog/how-to-create-server`).
- Внедрение Schema.org микроразметки `Article`.
- Добавление автоматического Оглавления и тегов для кросс-линковки.

---

## 🔒 Phase 9: Безопасность и Интеграция (Enterprise features)

### 3. Google 2FA (Двухфакторная Аутентификация)
Бескомпромиссная защита аккаунтов администраторов и владельцев крупных проектов, у которых на счету могут быть большие суммы баллов для бустов.
- Подключение TOTP алгоритмов (Google Authenticator / Authy).
- Генерация QR-кода в личном кабинете.
- Внедрение промежуточного шага ввода 6-значного кода при авторизации.

### 4. Завершение Публичного API
Предоставление разработчикам удобного интерфейса для парсинга нашего топа серверов.
- Добавление комплексного эндпоинта `GET /api/v1/servers` с пагинацией, фильтром по версиям и поиском.
- Жесткое лимитирование (Rate Limits) для защиты от парсеров, создающих сильную нагрузку на БД.
